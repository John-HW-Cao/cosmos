## ControlNet Architecture Comparison (Deep Dive)

### Transfer1 — Classic ControlNet

```
Control hint (latent space)
    │
    ▼
Control Hint MLP  [16 → 16 → 32 → 32 → 96 → 96 → 256 → 4096]  (SiLU)
    │
    ▼
ControlNet Encoder  (copy of first 14 DiT blocks)
  Block 0  ──zero_block_0──►  +residual to base DiT block 0
  Block 1  ──zero_block_1──►  +residual to base DiT block 1
  ...
  Block 13 ──zero_block_13──► +residual to base DiT block 13

MultiControlNet: run one encoder per modality → sum all residuals
```

### Transfer2.5 — VACE-style Control

```
Control hint + mask channel  (up to 641 ch)
    │
    ▼
VACE ControlEncoderDiTBlock at layer 0
  before_proj: Linear(641 → 2048)   ← block 0 only
  internal DiT block computation
  after_proj:  Linear(4×2048 → 2048) ← fuses 4 modality branches
    │
    ├──► inject residual into base DiT layer 0
    ▼
(pass through base DiT layers 1–6)
    ▼
VACE ControlEncoderDiTBlock at layer 7
  after_proj:  Linear(4×2048 → 2048)
    │──► inject residual into base DiT layer 7
    ▼
(pass through base DiT layers 8–13)
    ▼
VACE ControlEncoderDiTBlock at layer 14  ──► inject at layer 14
(pass through layers 15–20)
VACE ControlEncoderDiTBlock at layer 21  ──► inject at layer 21
(pass through layers 22–27)
    ▼
Unified output
```

Key differences:
- Transfer1 copies **the first half** of the base DiT (14 blocks); Transfer2.5 uses only **4 lightweight encoder blocks** uniformly spaced.
- Transfer1 encodes control hints through a deep MLP before they enter the ControlNet; Transfer2.5 directly concatenates the control latent in channel dimension (VACE-style), preserving spatial structure.
- Transfer2.5 uses a single `after_proj` linear layer per control injection point to fuse up to 4 simultaneous control branches, rather than summing separate full-encoder outputs.

## Image Conditioning (New in Transfer2.5)

Transfer2.5 introduces **SigLip2** as an image context encoder:

```
Reference image
    │
    ▼
SigLip2 Vision Encoder  (img_latent_dim=1024, dropout=0.5)
    │
    ▼
I2VCrossAttentionFull  (inside each DiT block)
  Separate Q/K/V projections: q_img, k_img, v_img
  + q_img_norm (RMSNorm)
    │
    ▼
Image cross-attention output added to each block's hidden state
```

Transfer1 has **no** image context encoder. Image-conditioned generation in Transfer1 is handled only via the `condition_video_input_mask` channel-concat mechanism.

## Summary of Key Improvements in Transfer2.5

1. **Smaller but smarter model**: 2B vs 7B parameters, using VACE-style sparse control injection instead of copying half the DiT.
2. **Better image context**: SigLip2 cross-attention enables true image-to-video conditioning absent from Transfer1.
3. **Richer text understanding**: Qwen2.5-VL-7B option for reason-enhanced text conditioning.
4. **More efficient control**: 4 VACE blocks vs 14 ControlNet encoder blocks; direct latent concat vs deep MLP encoding.
5. **Learnable positional embedding**: rope3d is now learnable, improving generalization.
6. **Sparse attention**: Reduces memory and compute vs full O(N²) attention.
7. **Multi-camera and robot variants**: Expanded from single-camera world-model to physical robot control scenarios.
8. **Spatiotemporal masks**: Per-modality binary masks allow fine-grained spatial control over where each conditioning signal applies.
9. **Lower VRAM**: ~65 GB vs ~80 GB for 720p inference on H100.
